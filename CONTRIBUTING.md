# コントリビューションガイド

Live Comment Hub への関心をありがとうございます。Issue / Pull Request を歓迎します。
このリポジトリは少し変わった運用をしているため、最初に読んでいただけると助かります。

## このリポジトリの位置づけ

このリポジトリは、アプリを**ビルドするために必要なソース一式**を公開したものです。
実際の開発は別の upstream リポジトリで行っており、リリースごとにこのリポジトリへ
エクスポートしています。そのため、いくつか通常の OSS と異なる点があります。

- 公開対象は「ビルドに必要な範囲」です。配布サーバの運用、内部設計ドキュメント、
  第三者から購入した素材などは含まれません。それらの領域への変更は受け取れません。
- コミット履歴はリリース単位でまとまっています（細かい開発履歴は upstream 側にあります）。

## ブランチ運用

| ブランチ | 役割 |
|---|---|
| `main` | リリース版。エクスポート専用で、ここへ直接 push / PR マージはしません。**ビルドする方はこのブランチを使ってください。** |
| `develop` | コントリビューション受け入れ用。**Pull Request はこのブランチ宛**に作成してください。 |

## Pull Request の流れ

1. `develop` から作業ブランチを切り、変更を加えてください。
2. `develop` 宛に Pull Request を作成してください。CI（ビルド + `npm test`）が走ります。
3. レビュー後、`develop` にマージします（あなたのコミットがそのまま残ります）。
4. 取り込まれた変更は upstream に反映され、**次のリリースで `main` に出ます**。
   リリース後、`develop` は `main` に再同期されます。
5. 反映されたコントリビューションは CHANGELOG / コントリビュータ一覧でクレジットします。

> `main` はエクスポートで再生成されるため、`main` 宛の PR や直接マージは取り込めません。
> 必ず `develop` 宛にお願いします。

## 開発環境

[README.md](README.md) の「必要環境 / ビルド / テスト」を参照してください。変更前後で
`npm test` が緑であることを確認してください。Rust を変更した場合は
`cargo check --manifest-path core/Cargo.toml` も実行してください。

## コーディング規約

- 既存のコードのスタイル・命名・パターンに合わせてください。
- 1 つの Pull Request は 1 つの目的に絞ってください。

## Issue

バグ報告・機能要望ともに歓迎します。再現手順・期待する挙動・実際の挙動を含めて
いただけると対応がスムーズです。
