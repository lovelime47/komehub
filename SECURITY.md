# セキュリティポリシー / Security Policy

## サポート対象 / Supported versions

セキュリティ修正は**最新リリース**に対して提供します。
Security fixes are provided for the **latest release** only.

## 脆弱性の報告 / Reporting a vulnerability

脆弱性を見つけた場合は、**公開 Issue には書かず**、非公開で報告してください。
If you find a vulnerability, please report it privately — **do not open a public issue**.

- GitHub の **「Report a vulnerability」**（リポジトリの Security タブ → Privately report a vulnerability）
- もしくは X の DM: [@lovelime47](https://x.com/lovelime47)

可能であれば、以下を含めてください:
Please include, if possible:

- 再現手順 / Steps to reproduce
- 影響範囲（何ができてしまうか）/ Impact
- 対象バージョン / Affected version

確認後、できるだけ早く修正し、次のリリースで配布します。報告いただいた方は、希望に応じて
リリースノートでクレジットします。
After triage, a fix will be shipped in a subsequent release. Reporters are credited in the
release notes on request.
