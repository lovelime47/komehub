structure / effects

一時効果を付ける

afterBindComment() は DOM bind 後に一時 class や timer を扱う場所です。

このサンプルで既にやっていること

afterBindComment: function (node, model, context) {
  if (!node || !model || !model.raw || !model.raw.amountDisplay) return;
  context.effects.addClass('paid-pulse', 'is-paid-pulse', {
    removeOnPhases: ['leaving']
  });
  context.effects.setTimeout('paid-pulse-timeout', 1000, function (effectContext) {
    if (effectContext.node && effectContext.node.classList) {
      effectContext.node.classList.remove('is-paid-pulse');
    }
  });
}

CSS 側の .comment.is-paid-pulse .card-shell が pulse animation を再生します。

例: メンバー加入も光らせる

if (model.raw.isMembership) {
  context.effects.addClass('member-pulse', 'is-member-pulse', {
    removeOnPhases: ['leaving']
  });
}
.comment.is-member-pulse .card-shell {
  animation: paid-pulse 900ms ease;
}

なぜ context.effects を使うか

コメントが削除されたときに class と timer を runtime の lifecycle に合わせて片付けられるためです。通常の setTimeout() だけで書くと、削除済み DOM を触る処理が残りやすくなります。
