元データを加工する

beforeCommitComment() は、HTML が読むための display.* を作る場所です。DOM は触りません。

このサンプルで作っている値

display.html

typing 用の HTML。本文が変わらない同一 id 更新では前回値を再利用します。

display.role

OWNER / MOD / MEMBER の小さな pill 表示です。

display.notice

スパチャやメンバー加入の強調文です。

display.themeKey

CSS の data-kh-theme と themeAssets に使います。

例: role 表示を変える

function roleLabel(rawComment) {
  if (rawComment.isOwner) return 'HOST';
  if (rawComment.isModerator) return 'STAFF';
  if (rawComment.isMember || rawComment.isMembership) return 'MEMBER';
  return '';
}

例: 長文コメントに notice を出す

function noticeLabel(rawComment) {
  if (rawComment.amountDisplay) return 'SUPER CHAT ' + rawComment.amountDisplay;
  if (Array.from(rawComment.comment || '').length > 80) return 'LONG COMMENT';
  if (rawComment.isMembershipGift) return 'MEMBERSHIP GIFT';
  if (rawComment.isMembership) return rawComment.membershipHeader || 'NEW MEMBER';
  return '';
}

prevModel を使う理由

同じ id のコメントが更新されたときに typing HTML と theme を維持するためです。これをしないと、既存カードが更新のたびに再アニメーションして見えます。
