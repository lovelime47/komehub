structure / themes

テーマを増やす

テーマは script.js、style.css、assets/、manifest.json の4か所を同じ key で揃えます。

テーマのつながり

THEME_LABELS['theme-sunset']
  -> display.themeKey
  -> data-kh-theme="theme-sunset"
  -> .comment[data-kh-theme="theme-sunset"]
  -> themeAssets['theme-sunset']

例: blue テーマを追加する

JS の label に追加

var THEME_LABELS = {
  'theme-teal': 'MINT',
  'theme-violet': 'VIOLET',
  'theme-sunset': 'SUNSET',
  'theme-blue': 'BLUE'
};

themeAssets に追加

'theme-blue': {
  '--theme-icon-image': 'assets/icon-blue.svg'
}

CSS に追加

.comment[data-kh-theme="theme-blue"] {
  --theme-a: #2563eb;
  --theme-b: #93c5fd;
  --theme-c: #eff6ff;
}

manifest に選択肢を追加

{ "value": "blue", "label": "ブルー" }

同じテーマを維持する理由

assignThemeKey() は prevModel があるとき前回 theme を維持します。同じコメントが更新されても色が変わらず、カードがちらつきにくくなります。
