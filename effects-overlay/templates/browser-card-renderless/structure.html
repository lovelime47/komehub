browser-card-renderless 構造解説

多層カード型コメントを HTML-first / renderless で作るための案内です。 HTML はカードの骨格、CSS は多層表現、JS は typing / theme / 一時効果の model 加工に集中します。

ファイルの役割 起動の流れ コメント表示の流れ 変更箇所マップ 分担ルール

ファイルの役割

index.html

#comments、コメント1件分の <template>、設定 JSON を持ちます。カードの DOM 階層を変えるときはここを見ます。

style.css

カード幅、左レール、背景レイヤー、テーマ色、typing、スパチャ pulse を受け持ちます。

script.js

display.html、display.role、display.notice、display.themeKey を作ります。DOM は作りません。

assets/

テーマごとの SVG icon です。themeAssets から CSS 変数 --theme-icon-image へ流れます。

起動の流れ

HTML-first starter を起動する

data-kh-start="list" と data-kh-template="#comment-template" により、runtime が list renderless として起動します。

設定 JSON を読み込む

cssVars、toggleCssVars、toggleClasses が manifest の UI key と CSS をつなぎます。

JS hook を登録する

beforeCommitComment() が表示用 model を作り、afterBindComment() が一時効果を付けます。

コメント1件が表示されるまで

raw comment

name、comment、amountDisplay、isMember などが入ります。

display model

beforeCommitComment() が typing HTML、role、notice、theme key、theme label を追加します。

template bind

data-kh="display.html" や data-kh="display.notice" に値が入り、root には data-kh-theme が付きます。

CSS 表示

CSS が data-kh-theme、.is-superchat、.typing-char を使ってカードを描画します。

rawComment.comment
  -> buildTypingHtml()
  -> display.html
  -> <div class="comment-text" data-kh="display.html" data-kh-mode="html">

変更箇所マップ

見た目を変えたい

カード幅、余白、角丸、左レール、typing、pulse を調整する。

表示項目を増やしたい

<template> に data-kh 要素を追加する。

元データを加工したい

beforeCommitComment() で role、notice、typing HTML を作る。

設定 UI を増やしたい

manifest.json の key を CSS 変数、class、callback へつなぐ。

テーマを増やしたい

THEME_LABELS、themeAssets、CSS の theme selector を揃える。

一時効果を付けたい

afterBindComment() と context.effects で lifecycle に乗せる。

分担ルール

HTML はカード構造

DOM 階層、表示順、data-kh の置き場所を決めます。

CSS は多層表現

背景、左レール、色、animation は CSS に寄せます。

JS は派生値

DOM を作らず、HTML が読むための display.* を作ります。

manifest は作者 UI

ユーザーに触らせたい設定だけを定義します。
