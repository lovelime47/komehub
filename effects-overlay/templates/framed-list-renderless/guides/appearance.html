structure / appearance

見た目を変える

色、余白、サイズ、角丸、影、タイピング、入退場アニメーションは基本的に style.css の責務です。HTML の構造や JS の model 加工を変えずに、まず CSS だけで調整します。

表示項目 元データ加工 設定 UI テーマ 一時効果

最初に触る場所

:root

テンプレート全体で使う基本値です。幅、高さ、透明度、アクセント色などをまとめて持っています。

.frame-shell

外枠です。サイズ、背景、角丸、影、固定高さを調整します。

.comment

コメント1件のカードです。余白、列構成、背景、枠線、影を調整します。

.typing-char

本文を1文字ずつ出す表現です。速度、上限、動きの方向を調整します。

例: フレームを大きくする

既定値を変えるだけなら :root を編集します。 ユーザー設定から変えたい場合は manifest.json と index.html の JSON config も確認します。 

:root {
  --panel-width: 560px;
  --frame-height: 720px;
}

注意

.frame-shell は固定高さです。コメントが増えても全体の高さを揺らさないためです。 height: auto にすると、コメント流入時に OBS 上の占有サイズが変わりやすくなります。

例: 角丸と影を弱くする

.frame-shell {
  border-radius: 4px;
  box-shadow: 0 10px 24px rgba(0, 0, 0, 0.22);
}

.comment {
  border-radius: 4px;
  box-shadow: none;
}

例: タイピングを調整する

260ms は1文字が出てくる動きの長さ、 18ms は次の文字を出し始める間隔、 40 は delay を増やし続ける上限です。 

.typing-char {
  animation: typing-in 260ms var(--kh-ease-out) forwards;
  animation-delay: calc(min(var(--char-index), 80) * 24ms);
}

この例では、80文字目まで順番に出し、1文字ごとの間隔を少し遅くしています。

例: 登場方向を変える

lifecycle class は runtime が付けます。CSS 側では「開始位置」「表示中」「退場中」を定義します。 

.comment.is-entering {
  opacity: 0;
  transform: translateX(18px);
}

.comment.is-active {
  opacity: 1;
  transform: translateX(0);
}

共通の仕組みやリファレンスを読むには: こめはぶ本体の「テンプレート管理 → テンプレート開発ガイドを開く」から アプリ内ヘルプを開いてください。テンプレート作成ガイド・API リファレンス・ わんコメ移植ガイド・チュートリアル例が読めます。
