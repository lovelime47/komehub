structure / effects

一時効果を付ける

コメント DOM に一時的な class や timer を付ける処理は afterBindComment() と context.effects に寄せます。 lifecycle に合わせて cleanup できるため、退場後の timer 残りを避けやすくなります。

このサンプルで既にやっていること

スパチャだけ is-paid-highlight を一時的に付けています。

afterBindComment: function (node, model, context) {
  if (!node || !model || !model.raw || !model.raw.amountDisplay) return;
  context.effects.addClass('paid-highlight', 'is-paid-highlight', {
    removeOnPhases: ['leaving']
  });
  context.effects.setTimeout('paid-highlight-timeout', 900, function (effectContext) {
    if (effectContext.node && effectContext.node.classList) {
      effectContext.node.classList.remove('is-paid-highlight');
    }
  });
}

例: メンバー加入を一瞬強調する

JS で class を付ける

if (model.raw.isMembership) {
  context.effects.addClass('membership-pulse', 'is-membership-pulse', {
    removeOnPhases: ['leaving']
  });
  context.effects.setTimeout('membership-pulse-timeout', 1200, function (effectContext) {
    effectContext.node.classList.remove('is-membership-pulse');
  });
}

CSS で見た目を作る

.comment.is-membership-pulse {
  box-shadow: 0 0 0 3px color-mix(in srgb, var(--kh-role-member) 45%, transparent);
}

なぜ普通の setTimeout() ではなく context.effects を使うか

コメントが消えたあとに timer が残ると、存在しない DOM を触ろうとしたり、 次のコメントに古い処理が影響したりします。 context.effects はコメント lifecycle と紐づいているので、退場時に cleanup しやすくなります。

使い分け

beforeCommitComment()

表示用の値を作る。DOM は触らない。

afterBindComment()

DOM に class を付ける。サイズ測定や一時効果を置く。

CSS

付けられた class がどう見えるかを定義する。
