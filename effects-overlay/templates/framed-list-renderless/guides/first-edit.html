最初の改造

こめはぶのテンプレートを初めて触る場合は、まず「どのファイルが何を担当するか」を体で確認します。 このページの順に、小さい変更から試してください。

4つのファイルの分担

ファイル担当初心者が最初に触る例
manifest.json アプリ設定画面に出る項目と既定値 headerText.default、typingSpeed.default
style.css 見た目、余白、色、アニメーション .comment、.comment-text、.typing-char
index.html 画面構造と、model の値を入れる場所 <template id="comment-template"> 内の表示順
script.js raw comment から表示用 model を作る beforeCommitComment() の display.prefix

最初にやると分かりやすい変更

タイトルを変える

manifest.json の headerText.default を変えます。

{
  "key": "headerText",
  "type": "text",
  "label": "ヘッダータイトル",
  "default": "My Chat"
}

コメントカードの背景を変える

style.css の .comment を変えます。

.comment {
  background: rgba(255, 255, 255, 0.12);
}

スパチャのラベル文言を変える

script.js の beforeCommitComment() 内を変えます。

if (rawComment.amountDisplay) {
  prefix = 'THANK YOU ' + rawComment.amountDisplay;
}

renderless とは

このテンプレートでは、JavaScript がコメント DOM を全部手作りするのではなく、 index.html の <template> を runtime が複製します。 その複製した HTML に、data-kh で指定した model の値を差し込みます。 この形を、このサンプルでは renderless 構成と呼んでいます。

data-kh と beforeCommitComment() の関係

rawComment.comment
  -> script.js の beforeCommitComment()
  -> display.html を作る
  -> index.html の data-kh="display.html" に入る
  -> style.css の .typing-char で見た目が付く

data-kh="name" は raw comment 由来の model.name を読みます。 data-kh="display.html" は beforeCommitComment() が作った model.display.html を読みます。

迷ったときの判断

迷い見る場所
見た目だけ変えたいstyle.css
設定画面に項目を出したいmanifest.json と index.html の cssVars / toggleClasses
表示する値を増やしたいscript.js の beforeCommitComment() と index.html の data-kh
一瞬だけ光らせたいscript.js の afterBindComment() と style.css
登場アニメーションを増やしたいmanifest.json、script.js の animationClasses、style.css の kh-anim-*
