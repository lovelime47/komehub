structure / model

元データを加工する

raw comment を表示用の形に変える場所は script.js の beforeCommitComment() です。ここでは DOM を触らず、HTML から参照する model の値だけを作ります。

責務

ここでやる

  • 表示ラベルを作る
  • 本文 HTML を作る
  • テーマ key を選ぶ
  • 長文、スパチャ、メンバーなどの派生情報を作る

ここではやらない

  • DOM を query して書き換える
  • timer を作る
  • 一時 class を付ける
  • CSS の値を直接変更する

例: スパチャ文言を変える

現在は SUPER CHAT 金額 という prefix を作っています。

if (rawComment.amountDisplay) {
  prefix = 'THANKS ' + rawComment.amountDisplay;
}

HTML 側では data-kh="display.prefix" の要素がこの値を表示します。

例: 名前表示を加工する

JS で派生値を作る

var nameLabel = rawComment.isOwner
  ? 'HOST ' + rawComment.name
  : rawComment.name;

return {
  display: {
    nameLabel: nameLabel,
    prefix: prefix,
    html: buildTypingHtml(rawComment, prevModel)
  }
};

HTML で読む path を変える

<span class="name" data-kh="display.nameLabel"></span>

例: 長文だけ印を付ける

renderless bind では class 付与まで自動化しない場合があります。 まず model に派生値を作り、HTML で表示するのが分かりやすいです。 

var plainText = rawComment.comment || '';
var longNotice = Array.from(plainText).length > 80 ? 'LONG' : '';

return {
  display: {
    longNotice: longNotice,
    prefix: prefix,
    html: buildTypingHtml(rawComment, prevModel)
  }
};
<span class="notice" data-kh="display.longNotice" data-kh-empty="hide"></span>

prevModel を使う理由

同じコメント id が更新されたとき、前回の model が prevModel に入ります。 このサンプルでは、本文が変わっていない場合に前回の display.html を再利用し、 typing animation が巻き戻らないようにしています。
