設定 UI を増やす

ユーザーがアプリ上で触る設定は manifest.json の uiSchema に定義します。 追加した key は、CSS 変数、toggle class、または JS callback のどれかへつなぎます。

3つのつなぎ方

CSS 変数へ流す

サイズ、色、透明度のように CSS だけで完結する設定向けです。

class を付け外しする

表示/非表示、密度変更、レイアウト切替のような ON/OFF 向けです。

JS callback で処理する

DOM の文字を変える、複数 class を切り替えるなど、少し処理が必要な設定向けです。

例: フレーム幅 (width) が CSS 変数へ流れる仕組み

表示位置 (左/右) と横幅 (width) は全テンプレ共通の設定として自動で付きます。このテンプレートは共通の width をフレーム幅として --panel-width に流しています。同じ要領で独自の slider を追加することもできます。

manifest.json で範囲を上書きする (任意)

{
  "key": "width",
  "type": "slider",
  "label": "フレーム幅",
  "min": 320,
  "max": 720,
  "step": 10,
  "default": 460,
  "suffix": "px"
}

index.html の JSON config で CSS 変数へ流す

"cssVars": {
  "width": ["--panel-width", "px"]
}

style.css で使う

.frame-shell {
  width: min(var(--panel-width), calc(100vw - 48px));
}

例: ヘッダー表示 toggle を追加する

{
  "key": "showHeader",
  "type": "toggle",
  "label": "ヘッダー表示",
  "default": true
}
"toggleClasses": {
  "showHeader": ["hide-header", "body", true]
}
body.hide-header .frame-header {
  display: none;
}

配列の最後の true は反転指定です。 showHeader が false のときに hide-header を付けます。

例: タイトル text を追加する

{
  "key": "headerText",
  "type": "text",
  "label": "ヘッダータイトル",
  "default": "Comment Board"
}
config: {
  callbacks: {
    headerText: function (value) {
      setText('.frame-title', value, 'Comment Board');
    }
  }
}

確認ポイント

  • manifest.json の key と index.html / script.js の key が一致しているか。
  • 既定値が manifest.json と preview の applyConfig で矛盾していないか。
  • JSON config 内にはコメントを書かない。説明は HTML コメント側に書く。

共通の仕組みやリファレンスを読むには: こめはぶ本体の「テンプレート管理 → テンプレート開発ガイドを開く」から アプリ内ヘルプを開いてください。テンプレート作成ガイド・API リファレンス・ わんコメ移植ガイド・チュートリアル例が読めます。
