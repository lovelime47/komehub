色プリセットを増やす

このテンプレートの「テーマ」は、コメントごとに変わるテーマではなく、 配信者が設定画面で選ぶ colorPreset です。 まずは manifest.json のプリセットを増やすのが一番安全です。

仕組み

manifest.json の colorPreset
  -> option.applyValues
  -> accentColor / textColor / bgColor などが更新される
  -> index.html の cssVars
  -> style.css の CSS 変数へ反映される

例: 桜プリセットを追加する

manifest.json の colorPreset.options に追加する

{
  "value": "sakura",
  "label": "桜",
  "applyValues": {
    "accentColor": "#f472b6",
    "textColor": "#fff7fb",
    "bgColor": "#1f0a16",
    "highlightColor": "#fde68a",
    "borderColor": "#ffffff"
  }
}

必要なら既定値にする

{
  "key": "colorPreset",
  "type": "buttons",
  "label": "色プリセット (選ぶと下の個別色を上書きします)",
  "default": "sakura"
}

個別色との関係

プリセットは accentColor、textColor、bgColor、 highlightColor、borderColor をまとめて入れるショートカットです。 配信者はプリセットを選んだあと、個別色をさらに調整できます。

失敗しやすい点

  • value は他の option と重複させません。
  • applyValues の key は、既存の色設定 key と同じ名前にします。
  • 背景色と本文色のコントラストが低いと、OBS 上で読みにくくなります。
  • コメントごとに色を変えたい場合は、このページの範囲を超えるため、まず guides/model.html で model の作り方を確認します。

共通の仕組みやリファレンスを読むには: こめはぶ本体の「テンプレート管理 → テンプレート開発ガイドを開く」から アプリ内ヘルプを開いてください。テンプレート作成ガイド・API リファレンス・ わんコメ移植ガイド・チュートリアル例が読めます。
