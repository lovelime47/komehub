framed-list-renderless 構造解説

この HTML は、サンプルフォルダだけを見て新しいテンプレートを作れるようにするための案内です。 メインドキュメントを読まなくても、HTML、CSS、JS、manifest、runtime の分担が分かることを目的にしています。

共通の仕組みやリファレンスを読むには: こめはぶ本体の「テンプレート管理 → テンプレート開発ガイドを開く」から アプリ内ヘルプを開いてください。テンプレート作成ガイド・API リファレンス・ わんコメ移植ガイド・チュートリアル例が読めます。本ページはこのサンプル固有の構造解説です。

ファイルの役割

index.html

画面の骨格です。フレーム、ヘッダー、フッター、コメント一覧の差し込み先、 コメント1件分の <template>、HTML-first 用の JSON 設定を持ちます。

style.css

見た目の責務を持ちます。固定高さ、余白、色、テーマ、lifecycle animation、 typing animation、role class の色分けをここで調整します。

script.js

表示用 model を作ります。DOM を組み立てるのではなく、 beforeCommitComment() で display.html や display.prefix などを作ります。

manifest.json

テンプレートの登録情報と設定 UI の定義です。ここにある key が、 index.html の JSON config や script.js の callbacks と対応します。

起動の流れ

1

HTML 読み込み

index.html が読み込まれ、#comments と #comment-template が用意されます。

2

CSS 適用

style.css が runtime 共通 CSS とテンプレート固有 CSS を読みます。

3

JSON 設定

#komehub-template-options の JSON が HTML-first 設定として読まれます。

4

runtime 起動

htmlFirst.start() が data-kh-start="list" を list starter として初期化します。

5

コメント待機

runtime がコメントを受け取り、template へ値を bind できる状態になります。

このサンプルの重要点

renderComment() で HTML 文字列を丸ごと作るのではなく、 HTML の <template> と data-kh に runtime が値を差し込みます。 そのため、HTML と CSS を見れば構造と見た目を追いやすくなります。

コメント1件が表示されるまで

1

raw comment

name、comment、amountDisplay などの元データが入ります。

2

model 加工

raw comment に、beforeCommitComment() が作る display.prefix、display.html を足したものが model です。

3

template 複製

runtime が #comment-template を複製し、コメント1件分の DOM を作ります。

4

data-kh bind

data-kh="name" は model.name、data-kh="display.html" は model.display.html を HTML に入れます。

5

表示と効果

lifecycle class と afterBindComment() により、登場 animation や highlight が動きます。

rawComment.comment
  -> beforeCommitComment()
  -> display.html
  -> <div class="comment-text" data-kh="display.html" data-kh-mode="html">
  -> .typing-char / @keyframes typing-in

変更箇所マップ

具体的な作業手順は専用ページに分けています。 まず目的に近いページを開き、そこにある手順とコード例を見ながら編集してください。

最初の改造を試したい

タイトル、色、カード、本文、タイピングのような安全に触れる場所から始める。

配信者向けに調整したい

OBS 推奨サイズ、表示位置、色、文字サイズ、タイピング演出の実用設定を見る。

見た目を変えたい

色、余白、フォント、フレーム高さ、typing、入退場 animation を調整する。

表示項目を増やしたい

<template> に要素を追加し、data-kh で model の値を表示する。

元データを加工したい

beforeCommitComment() で raw comment から表示用 model を作る。

設定 UI を増やしたい

manifest.json の uiSchema から CSS 変数、class、callback へつなぐ。

色プリセットを増やしたい

manifest.json の colorPreset に候補を追加し、既存の色設定へ反映する。

一時効果を付けたい

afterBindComment() と context.effects で lifecycle に乗る効果を作る。

分担ルール

HTML は構造

何をどの順番で表示するかを決めます。 data-kh は model の値を差し込む目印です。

CSS は見た目

配置、色、余白、サイズ、animation を受け持ちます。 コメント流入時に高さを揺らしたくない場合は、固定高さの考え方を崩さないようにします。

JS は model と効果

beforeCommitComment() では DOM を触らず model を作ります。 DOM へ一時的な class や timer を付ける処理は afterBindComment() に寄せます。

manifest は作者 UI

ユーザーに触らせたい設定だけを定義します。 設定 key は CSS 変数、toggle class、callback のどれかへ明確につなぎます。
