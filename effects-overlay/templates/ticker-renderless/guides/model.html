元データを加工する

このサンプルは最小構成のため beforeCommitComment() を使っていません。標準 model にない表示用値が必要なときだけ追加します。

まず直接読める値を確認する

name、text、amount、membershipHeader、badge、sticker は直接 data-kh で読めます。

例: 独自の短いラベルを作る

runtime.htmlFirst.start({
  mode: 'ticker',
  renderlessModel: true,
  beforeCommitComment: function (rawComment) {
    return {
      display: {
        shortKind: rawComment.amountDisplay ? 'SC' : ''
      }
    };
  }
});
<span class="kind" data-kh="display.shortKind" data-kh-empty="hide"></span>

注意点

  • 既存 field を別名に詰め直すだけなら beforeCommitComment() は不要です。
  • 横幅が変わる派生表示を追加したら、設定変更時に recalcTicker() が必要になることがあります。
  • DOM を触る処理は beforeCommitComment() ではなく afterBindComment() に寄せます。
