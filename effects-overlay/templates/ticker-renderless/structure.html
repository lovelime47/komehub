ticker-renderless 構造解説

横流れコメントを HTML-first / renderless で作るための案内です。 list と違い、コメントを積むのではなく #track 全体を transform で左へ動かします。

ファイルの役割 起動の流れ コメント表示の流れ 変更箇所マップ 分担ルール

ファイルの役割

index.html

#comments、横移動する #track、コメント1件分の <template>、設定 JSON を持ちます。

style.css

グローテキスト、名前位置、スパチャ強調、#track の transition を受け持ちます。

script.js

横流れ starter を起動し、縦位置、名前位置、光彩、速度変更時に runtime へ再計算を依頼します。

manifest.json

文字サイズ、名前表示、光彩色、速度など、横流れ作者が触りたい設定 UI を定義します。

起動の流れ

ticker starter を選ぶ

data-kh-start="ticker" と mode: 'ticker' により、runtime は横流れ starter として起動します。

renderless model を有効にする

renderlessModel: true により、renderComment() ではなく <template> と data-kh で描画します。

track を動かす

コメント追加時に runtime が #track の幅と位置を計算し、CSS transition で左へ動かします。

コメント1件が表示されるまで

template 複製

#ticker-item-template から .comment が複製されます。

data-kh bind

name、membershipHeader、badge、amount、text、sticker が差し込まれます。

track に追加

新しいコメントが #track の右側へ入り、track 全体の移動距離が更新されます。

古いコメントを整理

画面外へ出たコメントから削除されます。左端に見えているコメントを急に消さないことが重要です。

new comment
  -> <template id="ticker-item-template">
  -> #track.appendChild(comment)
  -> #track.style.transform = translateX(...)
  -> transition: transform var(--anim-duration)

変更箇所マップ

見た目を変えたい

グロー、文字サイズ、名前位置、横間隔、スパチャ強調を調整する。

表示項目を増やしたい

<template> に data-kh 要素を追加する。

元データを加工したい

必要なときだけ beforeCommitComment() を追加する。

設定 UI を増やしたい

CSS 変数、toggle class、callback のどれへつなぐか決める。

グロー色を増やしたい

GLOW_COLORS、manifest の options、callback を揃える。

横流れの動きを変えたい

速度、再計算、overflow trim、track transition の注意点を見る。

分担ルール

HTML は1件分の形

横流れでも1コメント分の DOM は <template> に置きます。

CSS は光彩と transition

見た目と #track の移動 transition を受け持ちます。

JS は設定反映

DOM を作らず、runtime の ticker helper へ再計算や速度変更を依頼します。

runtime は移動管理

コメント追加、track の transform、画面外コメントの整理を担当します。
