リスナー判定

経過日数 + 最近の活動 で 6 ランクに分類
プリセット:

① 経過日数の境界

各リスナーの「初コメから今までの日数」でランクを決めます。

X 日 〜 Y 日 = 常連候補。 X < Y で設定 (= 自動補正あり)。

② 活動中の判定

「常連 / 古参 候補」のうち、 実際に 常連 / 古参 と認めるかは 「最近来てるか」で判定します。

活動中の条件を 満たす → 期間に応じて 常連 / 古参。
活動中の条件を 満たさない + 直近 N+1 枠 (= 過去 N 枠 + 最終枠) のどこかで発言あり → 復帰。
活動中の条件を 満たさない + 直近 N+1 枠で発言ゼロ → 離脱。

プレビュー —

リスナーランク一覧

配信詳細・コメント検索の chip で使われる 6 分類 (= per-listener、 静的)
  • 新規 この配信で初めてコメントした人 (連投も含む)
  • 新参 過去枠で初コメ済 + 初コメから X 日以内
  • 常連 初コメから X 日以上 〜 Y 日未満 + 活動中 (= 直近 N 枠中 M 枠以上)
  • 古参 初コメから Y 日以上 + 活動中 (= 直近 N 枠中 M 枠以上)
  • 復帰 初コメから X 日以上 + 活動中ではない + 直近 N+1 枠 (= 過去 N 枠 + 最終枠) のどこかで発言あり
  • 離脱 初コメから X 日以上 + 直近 N+1 枠で発言ゼロ (= 完全に来なくなった)

コメントタグ一覧

チャット欄に出る 4 種のタグ (= per-comment、 配信初発言で 1 度だけ付く)

ランクは その人がどんなリスナーか を表すのに対し、 タグは この発言がこの配信での何回目か / どんな登場か を表します。 タグは各リスナーの「この配信での初発言」に 1 度だけ付き、 2 回目以降のコメには付きません。

  • 初見 全期間で初めてコメントした人 (= ハブが初めて観測)
  • 再訪 過去枠で初コメ済 + 初コメから X 日以内 (= 新参の人がまた来てくれた)
  • 今北 過去枠で初コメ済 + 活動中 (= 直近 N 枠中 M 枠以上) — 常連 / 古参 が顔出し
  • 帰還 過去枠で初コメ済 + 初コメから X 日以上 + 直近 N 枠で発言ゼロ (= 一旦離脱した人が久々登場)

データ・バックアップ

データ管理 (インポート / エクスポート)

リスナー / コメントの JSONL 入出力、わんコメ DB との連携、双方向同期。

こめはぶ形式 (JSON Lines) エクスポート

バックアップ用にすべてのリスナー / 配信 / コメントを 1 ファイルに書き出します (1 行 = 1 レコード)。
ファイルが大きくなる場合があります。

こめはぶ形式 (JSON Lines) インポート

他環境のエクスポートファイルを取り込みます。
同じファイルを複数回読み込んでも結果は変わりません (冪等)。

わんコメからインポート

わんコメのデータフォルダ (%APPDATA%\onecomme\ など) を選択して、 comments.db と onecomme.db から取り込みます。
自チャンネル設定が必要。自チャンネル外の配信のコメントは自動で除外されます。
⚠ わんコメを終了してから実行してください (起動中だと取得結果が不安定になります)。

わんコメへ書き戻し (Plan A)

こめはぶで蓄積したリスナー / コメントをわんコメ DB に書き戻します。
書き戻し前に自動で data/onecomme-backups/ へバックアップを取ります。
⚠ わんコメ起動中・配信中は実行できません。先に「わんコメからインポート」を 1 度実行してスキーマハッシュを記録しておいてください。

今すぐ同期 (双方向)

わんコメ → こめはぶ → わんコメ を一気通貫で実行します (F-21)。
配信中・わんコメ起動中・自チャンネル未設定はスキップされます。
こめはぶ起動時にも自動でこの同期が走ります (F-20)。
