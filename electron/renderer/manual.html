Live Comment Hub / こめはぶ

配信者向けヘルプ

こめはぶは、YouTube Live のコメントを取得し、OBS やブラウザにコメント表示テンプレートとコメント連動演出を出すためのツールです。 このヘルプでは、配信で使うための操作だけを順に説明します。

目次

1. まず使ってみる 2. OBS / ブラウザ表示 3. コメント表示テンプレート 4. コメント連動演出 5. コメント読み上げ 6. スマホ操作 7. リスナー管理 8. シーン運用 9. 困ったとき

1. まず使ってみる

初回は「準備」から始める

  1. 右上の 準備 を開きます。
  2. 現在ライブ中の YouTube 配信 URL を貼り付けて、接続 を押します。自分の配信でなくても動作確認できます。
  3. コメントがアプリ内に流れることを確認します。
  4. 準備の各ステージ (接続 / OBS 表示 / リスナー履歴) を順に確認します。

自チャンネル設定 (初見・常連判定を使う場合)

初見歓迎演出やリスナー管理を使う場合は、自分の YouTube チャンネル ID (UC... 形式) を登録します。準備 の Stage 3、またはリスナー管理画面の上部から登録できます。

チャンネルが複数ある場合は複数登録できます。サブチャンネルでの配信も自分の配信として扱われます。

コメント取得だけなら YouTube ログインは基本的に不要です。

2. OBS / ブラウザに表示する

OBS に入れる

  1. 準備 の OBS に表示 を開くか、メイン画面の各 URL を確認します。
  2. コメント表示 URL を OBS のブラウザソースへドラッグアンドドロップします。
  3. 続けて、コメント連動演出 URL も同じ手順で別のブラウザソースとして追加します。
  4. または各 URL をクリックでコピーし、OBS のブラウザソースに貼り付けます。
  5. OBS 側で表示位置とサイズを調整します。

OBS を使わずブラウザで確認する

各 URL の隣にある 開く を押すと、同じ URL を既定ブラウザで確認できます。 配信に接続していない状態では実コメントが出ないため、先に配信 URL へ接続してください。

OBS でソースをドラッグして拡大すると、文字も画像もまとめて拡大されます。文字サイズを変えたい場合は、テンプレート設定側の文字サイズを調整します。

3. コメント表示テンプレート

テンプレートを選ぶ

  1. メイン画面の コメント表示テンプレート を開きます。
  2. テンプレートを追加 を押し、現在のシーンに使うテンプレートをいくつか追加します。
  3. 追加したテンプレートのうち 1 つをラジオボタンで選択します。選んだものが、そのシーンの配信用コメント表示になります。
  4. 複数のテンプレートを追加しておくと、配信中に切り替えて使えます。

見た目を調整する

各テンプレートの 設定 で、色、文字サイズ、表示件数、速度、背景などを調整します。 設定は自動保存され、すぐ表示に反映されます。

動作を確認する

設定画面の 確認用に開く でブラウザ表示を確認できます。URL はクリックでコピー、ドラッグで OBS へ追加できます。 実コメントを待たずに動きを試したい場合は、メイン画面の テストコメント でダミーコメントを送れます。

各テンプレ設定の 開発者向け は、HTML / CSS / JS を編集してテンプレート自体を作る人のための項目です。配信で使うだけなら通常は開かなくて構いません。

4. コメント連動演出

演出を使う

  1. シーンを選び、コメント連動演出 を開きます。
  2. 使いたい演出を ON にします。
  3. キーワード、スパチャ条件、リスナー条件、素材、効果音などを調整します。
  4. テスト で配信前に表示を確認します。

リスナー条件の用語

コメント連動演出の リスナー条件 で使う 4 種類の判定です。 自チャンネル設定が必要です (= 過去配信履歴を元に判定するため)。

  • 初見: 自分のチャンネルに 初めて コメントしたリスナー。サブチャンネルへの過去コメは含めて判定します。
  • 再訪: 常連以外で、この配信に 初めて コメントした状態。「お久しぶり」演出に向きます。
  • 常連: 自分の直近 10 配信のうち 3 枠以上でコメントしたリスナー。
  • 今北: 常連リスナーが、この配信に 初めて コメントした状態。常連の入場演出に向きます。

常連の判定基準 (= 直近 N 配信中 M 枠) は、設定モーダルの リスナー分類 から変更できます。

出すぎるときの調整

  • 特定の演出が多すぎる場合は、その演出のクールダウンを長くします。
  • 同じ人の連投で出すぎる場合は、全体設定の同一ユーザー間隔を長くします。
  • 画面が重い場合は、最大同時演出数を下げます。
  • 急いで止めたいときは、サイドバーの 演出全停止 を使います。

ポン出し

ポン出しバーに表示 を ON にした演出は、メイン画面のポン出しバーから手動で出せます。 コメント条件を待たずに出したい拍手、クラッカー、紙吹雪などに向いています。

配信中に手元で操作したい場合は、メイン画面の ポン出し から専用ウィンドウを開きます。 専用ウィンドウにはポン出し素材、演出の停止 / 再開、クリア、TTS の即時操作がまとまっています。

5. コメント読み上げ

はじめに

流れてきたコメントを音声で読み上げる機能です。読み上げソフトとしては 内蔵 (Windows 標準)、棒読みちゃん、 VOICEVOX のいずれかを選べます。 メイン画面の コメント読み上げ カードから操作します。

有効化 / 一時停止 / クリア / テスト

  • ON / OFF: 読み上げの有効・無効を切り替えます。
  • 一時停止 / 再開: 一時停止中の通常コメントは破棄されます (= 後から読み上げない)。スパチャ・メンバーシップは初期設定では待機キューに残ります。
  • クリア: 待機中のコメントと再生中の音声をすぐに止めます。
  • テスト: サンプル文を再生して動作を確認できます。

ソフトを選ぶ

設定画面の「出力先ソフト」から選びます。各ソフトの特徴は次の通りです。

  • 内蔵: Windows の標準音声合成。追加ソフト不要で、すぐ使えます。
  • 棒読みちゃん: 棒読みちゃんを別途インストール・起動しておく必要があります。音量・速度などはハブ側でも上書き可能です。
  • VOICEVOX: VOICEVOX エンジンを別途起動しておく必要があります。キャラクターとスタイル (ノーマル / あまあま 等) を選べます。

選んだソフトが起動していない場合は、メイン画面の コメント読み上げ バッジが赤色で VOICEVOX 未起動 のように表示されます。 ソフトを起動し直すと 10 秒以内に自動で復帰します。

出力デバイスの選び方

共通設定の 出力デバイス で読み上げ音声の出力先を指定できます。 内蔵と VOICEVOX に適用されます (棒読みちゃんは棒読み側で設定してください)。

  • システム既定: Windows のサウンド設定で既定になっているデバイスへ流します。何も指定しない場合のデフォルトです。
  • 配信に乗せたい場合: VB-Audio Cable Input 等の仮想ケーブルを選び、OBS の音声入力で同じケーブルを拾います。
  • 配信者だけ聞きたい場合: 物理ヘッドホン (例: PHL 288E2 等) を選びます。

一部のデバイス (キャプチャボード等) は SAPI 経由では再生できません。 テスト再生で「クラスが登録されていません」のようなエラーが出る場合は別のデバイスを試してください。 配信に乗せる用途なら VB-Audio Cable Input を強く推奨します。

ドロップダウンを開く瞬間にデバイス一覧を更新します。デバイスを後から接続した場合は一旦閉じて開き直してください。

読み上げ対象を絞る

共通設定で読み上げ対象 (通常コメント / スパチャ / メンバーシップ / ギフト) を切り替えられます。 投稿者ロール (オーナー / モデレーター / メンバー) も個別に ON / OFF できます。

  • 初期設定では オーナー (= 自分) のコメントは読み上げない 動作です。
  • 長すぎるコメントは 最大文字数 で打ち切られます (初期値 120 文字)。
  • 絵文字・URL を読み上げるかどうかも個別に設定できます。

困ったとき

  • 赤いバッジで「未起動」と出る: 選んだソフト (VOICEVOX / 棒読み) を起動してください。バッジ自動復帰まで最大 10 秒。
  • テストは鳴るが、配信に乗らない: 出力デバイスがヘッドホン等になっていないか確認。配信に乗せるなら VB-Audio Cable Input を選んでください。
  • 特定のコメントだけ読まれない: 共通設定の読み上げ対象 (通常 / スパチャ等) と最大文字数を確認。長文は途中で打ち切られます。
  • 連投で読み上げが追いつかない: クリア ボタンで待機キューを空にできます。

6. スマホ操作

スマホでポン出しを操作する

  1. メイン画面から ポン出し ウィンドウを開きます。
  2. スマホで開く を押します。
  3. 初回は確認画面を読み、問題なければ 有効にしてQR表示 を押します。
  4. 表示された QR コードをスマホで読み取ります。
  5. URL を手で送りたい場合は、URL 表示をタップするとクリップボードにコピーできます。

2 回目以降に確認を省きたい場合は、確認画面で 次からこの確認を表示しない を ON にしてから有効化してください。

リモートでできること

  • ポン出し素材を実行する
  • 演出を停止 / 再開する
  • 表示中演出と待機キューをクリアする
  • TTS を ON / OFF、一時停止、クリアする
  • 全画面ボタンでスマホ表示を広く使う
  • 現枠のコメント / リスナーを閲覧する (= 後述「スマホで現枠を見る」)
  • コメント検索結果を見る
  • 「コメント対応済み」「リスナー対応済み」「コメ非表示 / リスナー非表示」のトグルを切り替える

スマホ操作はリモート専用ポートで開きます。通常の管理機能、削除、初期化、インポート / エクスポート、テンプレート編集はスマホ側には公開しません。

スマホで現枠を見る

ポン出しウィンドウからリモートを有効化すると、ポン出し以外にも コメント / リスナーの閲覧画面がスマホで開けます。 配信中、PC のメイン画面を見続けずに手元で内容を確認したい時に使います。

  • /remote/ ホーム画面: 現枠のステータスと各画面への入口
  • /remote/comments: 流れている現枠コメントの一覧
  • /remote/listeners: 現枠で発言したリスナーの一覧
  • /remote/listeners/{ch}: リスナーごとの詳細 (過去コメ / SC 累計 / 配信ログ)
  • /remote/search: コメント検索結果 (= リスナー詳細から遷移)

いずれの画面も「読む + 軽い操作」用途で、文字入力を伴うタグ付けやニックネーム編集はできません。 メモ書きや編集は PC 側のリスナー管理画面で行ってください。

Windows セキュリティの警告が出たら

スマホから同じ Wi-Fi / LAN 経由で接続するため、Windows セキュリティが通信許可を確認することがあります。 表示された場合は プライベート ネットワーク を許可してください。

カフェや公共 Wi-Fi などのパブリック ネットワークでは許可しないでください。

7. リスナー管理

できること

  • リスナーごとのコメント履歴を確認できます。
  • 初見、再訪、今北、常連などの判定に使われます。
  • コメント連動演出の条件として、初見歓迎などに利用できます。

メイン画面の リスナー管理を開く から専用画面に入ります。一覧、配信ログ、コメント検索、インポート / エクスポートのタブがあります。

配信中の手元操作 (対応済み / 非表示 トグル)

配信中の対応漏れ防止 / 視認したくないリスナー対策のため、 コメント単位 / リスナー単位で 4 つのトグルが用意されています。 PC のメイン UI とスマホのリモート画面の両方で使えます。

  • 対応済み (コメ単位): 質問やリクエストへの反応漏れを防ぐ。 対応した時点でチェックを入れると、コメント検索の 未対応のみ で残りを絞り込めます。
  • 対応済み (リスナー × 配信枠): その枠でそのリスナーに 対応した記録。同じリスナーが翌枠に来た時はリセットされ、対応漏れを 防ぎます。
  • コメ非表示 (リスナー単位、累計): 指定リスナーのコメを 管理 UI のコメリストから隠します。
  • リスナー非表示 (リスナー単位、累計): 指定リスナーを リスナーリストから隠します。

切替は即時反映 + 5 秒間 元に戻す ボタンが画面下に出ます (Snackbar Undo)。 誤タップを防ぐため、コメ非表示 / リスナー非表示の ON / OFF はリスナー詳細の ⋯ メニューからの二段確認に変えています。

コメ非表示 / リスナー非表示は 配信者の UI 視認だけ 抑制します。 演出 / OBS テンプレ / TTS / コメント配信は通常通り動作するため、 相手にブロックや BAN として気付かれません。

わんコメからインポート

  1. リスナー管理の インポート / エクスポート タブを開きます。
  2. わんコメを終了してから、わんコメからインポート を選び、わんコメフォルダを指定します。
  3. 自分のチャンネル設定を基準に、自チャンネル配信ぶんのリスナー履歴を取り込みます。

わんコメ起動中は取得結果が不安定になります。先にわんコメを終了してください。

わんコメへ書き戻し / 自動同期

こめはぶで蓄積したリスナー / コメントは わんコメへ書き戻し でわんコメ DB に反映できます。 初回はインポートを先に 1 度実行し、スキーマハッシュを記録しておく必要があります。

設定モーダルの 自動同期 を ON にしておくと、ハブの起動時にわんコメから取り込み、 ハブの終了時にわんコメへ書き戻しが自動で実行されます。日常運用は自動同期に任せられます。

わんコメ起動中・配信中の書き戻しはスキップされます (DB 競合防止)。

コメント検索

リスナー管理の コメント検索 タブから、過去のコメントを横断検索できます。 配信中に「あの話どこかで聞いたな」をすぐ拾うのにも使えます。

  • キーワード: コメ本文の部分一致
  • リスナー名 / channel id: 特定の人だけに絞る
  • 期間: プリセット (今日 / 直近 7 日 / 30 日 等) または日付指定
  • コメント種別: 通常 / スパチャ / メンバー / ギフト など
  • タグ: 自分で付けたリスナータグ / 配信タグ
  • 未対応のみ: 対応済み 未チェックのコメだけ

よく使う組み合わせは 保存検索 として保存できます。 保存検索は左サイドから 1 タップで呼び出せます。

配信ログ / 配信詳細

リスナー管理の 配信ログ タブで、過去配信の一覧が見られます。 コメ数 / SC 累計 / 視聴ピーク / いいね数 でソートできます。

配信行をクリックすると、その配信の 詳細モーダル が開きます。 詳細モーダルには次が含まれます:

  • サムネ + タイトル + 配信時刻 + KPI (= コメ / SC / 視聴ピーク / 平均同接 / エンゲージ率)
  • 時系列ヒートマップ (= bin 幅は配信長に応じて自動調整)
  • 頻出語ランキング (= 一般語 stopword 除外済)
  • その枠で発言したリスナー一覧 + 各リスナーの活動 heatmap
  • コメント検索 (= 当該配信内に絞った検索)

8. シーン運用

シーンとは

シーンは、配信内容ごとのコメント表示テンプレートと演出セットです。 雑談、歌枠、ゲーム配信を分けておくと、OBS 表示や演出の切り替えが楽になります。

おすすめの使い分け

  • 雑談: コメント表示を読みやすくし、初見歓迎や軽いリアクション演出を使う。
  • 歌枠: 横流れコメントや拍手、ペンライト系の演出を使う。
  • ゲーム: 画面を邪魔しない小さめのテンプレートと控えめな演出にする。

9. 困ったとき

コメントが出ない

  • 配信 URL に接続できているか確認します。
  • 接続後にコメントが実際に投稿されているか確認します。
  • テンプレートがシーンに追加され、ラジオボタンで選択されているか確認します。
  • OBS ではなくブラウザで URL を開き、表示されるか確認します。
  • 動作だけ試したい場合は、テストコメント でダミーを送って確認します。

演出が出ない

  • コメント連動演出が ON になっているか確認します。
  • 演出ごとの条件にコメントが一致しているか確認します。
  • テストボタンで表示できるか確認します。
  • 全体設定の最大同時演出数やクールダウンで待機していないか確認します。

表示が大きい / 小さい

  • 文字サイズはテンプレート設定で調整します。
  • OBS のブラウザソースの幅と高さを確認します。
  • OBS 上でドラッグ拡大した場合は、全体が画像として拡大されます。

スマホで開けない

  • PC とスマホが同じ Wi-Fi / LAN に接続されているか確認します。
  • QR 画面の URL をタップしてコピーし、スマホのブラウザへ貼り付けて試します。
  • Windows セキュリティの警告で 許可しない を押した場合は、Windows の ファイアウォールによるアプリケーションの許可 から Live Comment Hub、開発中は Electron / node.exe / komehub_core の プライベート を許可します。
  • スマホ側の VPN、ゲスト Wi-Fi、AP 分離が有効な場合は、同じ Wi-Fi でも PC に届かないことがあります。

すべての設定を初期化 は強い操作です。実行前に退避が作成されますが、通常の調整では使わないでください。
